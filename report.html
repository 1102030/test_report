在线问答系统测试报告

项目描述

该项目是一个基于 Node.js + Express + MongoDB 的在线问答系统，支持用户注册、提问、回答、评论等功能。系统采用 RESTful API 提供前后端交互，用户可以提问并对其他用户的回答进行评论和点赞。

技术栈

  • 后端：Node.js, Express, MongoDB
  • 前端：React, Bootstrap
  • 测试工具：Postman, JMeter, Mocha, Chai

测试用例设计

编号 测试项 测试输入 预期输出 结果
TC001 用户注册 提供正确的用户名、邮箱、密码 用户注册成功 通过
TC002 用户登录 输入有效的用户名和密码 登录成功 通过
TC003 提问功能 输入有效问题并提交 问题成功发布 通过
TC004 回答功能 输入有效回答并提交 回答成功发布 通过
TC005 评论功能 输入有效评论并提交 评论成功发布 通过

性能测试

并发测试：使用 JMeter 模拟 500+ 并发用户同时提问和回答，检查系统的响应时间及稳定性。

响应时间测试：测试用户提问、回答和评论的平均响应时间，确保每个操作的响应时间不超过 500ms。

安全测试

  • SQL 注入测试：通过输入恶意 SQL 查询语句（如 ’ OR 1=1 --）检查是否能绕过身份验证。
  • XSS 攻击测试：测试是否存在脚本注入漏洞，防止攻击者执行 JavaScript 代码。
  • 权限测试：确保普通用户无法访问管理员权限接口。

测试总结

功能测试通过率：98%（共 15 个功能测试用例，1 个失败）

性能测试通过率：90%（响应时间大于 500ms 的请求为 10%）

发现缺陷数：3 个（1 个高优先级，2 个中优先级）

优化建议：

  • 增加 API 缓存机制，优化高并发情况下的响应时间。
  • 改进数据库索引，提升查询效率。
  • 增强前端表单验证，防止恶意数据提交。
